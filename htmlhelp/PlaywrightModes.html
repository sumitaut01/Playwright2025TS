Playwright Run Modes & Recording Guide

Understand how to run and record tests in Playwright

1. Headless vs Headed Mode

Headless: Runs without UI (fast, used in CI/CD)

npx playwright test

Headed: Runs with visible browser

npx playwright test --headed

2. UI Mode (Interactive)

Graphical interface to run and debug tests.

npx playwright test --ui

3. Debug Mode

Step-by-step execution with inspector.

npx playwright test --debug

4. Trace Mode

Records execution for debugging later.

npx playwright test --trace on

5. Parallel vs Serial

Run tests concurrently or one by one.

// Serial mode
test.describe.configure({ mode: 'serial' });

6. Project Execution

Run tests on different browsers.

npx playwright test --project=chromium
npx playwright test --project=firefox

🎥 7. Recording Mode (Codegen)

This is the mode used to record your actions and generate test code.

Start recording:

npx playwright codegen

Open a specific website:

npx playwright codegen https://example.com

Save recorded script:

npx playwright codegen -o test.spec.js

What Codegen Does:

  • Opens a browser window
  • Records clicks, typing, and navigation
  • Generates Playwright test code automatically

Best Use Cases:

  • Learning Playwright
  • Quick test creation
  • Finding selectors easily
Tip: Always clean up generated code before using it in production tests.
